# SignalMate 화면 구조서

작성일: 2026-03-27

## 1. 문서 목적

이 문서는 SignalMate MVP의 주요 화면과 화면별 핵심 요소, 사용자 액션, UX 우선순위를 정리한 구조서다.

목표는 `빠르게 만들 수 있는 MVP`와 `신뢰를 주는 결과 화면`을 만드는 것이다.

## 2. 전체 사용자 플로우

1. 랜딩페이지 유입
2. 분석 시작
3. 채팅 입력
4. 상황 선택
5. 분석 중 로딩
6. 결과 확인
7. 다음 메시지 추천 확인
8. 저장 또는 결제
9. 재분석 또는 종료

## 3. 화면 목록

- 랜딩페이지
- 회원가입 / 로그인
- 온보딩
- 채팅 입력 화면
- 상황 선택 화면
- 분석 로딩 화면
- 분석 결과 요약 화면
- 근거 상세 화면
- 다음 메시지 추천 화면
- 저장 분석 목록 화면
- 결제 / 업그레이드 화면
- 설정 / 개인정보 화면

## 4. 화면별 상세

## 4-1. 랜딩페이지

### 목적

문제 공감, 가치 이해, 분석 시작 전환

### 핵심 요소

- 메인 헤드라인
- 서비스 설명
- 공감 사례
- 기능 요약
- 결과 예시
- 신뢰 문구
- CTA 버튼

### 주요 액션

- 채팅 분석 시작하기
- 대기자 등록하기

### UX 메모

- 첫 화면에서 `예언 서비스`처럼 보이지 않게 해야 한다
- 개인정보 저장 없이 분석 가능하다는 문구가 중요하다

## 4-2. 회원가입 / 로그인

### 목적

분석 저장, 결제, 재방문 관리

### 핵심 요소

- 이메일 로그인 또는 소셜 로그인
- 비회원 체험 시작 옵션
- 개인정보 처리 안내

### 주요 액션

- 로그인
- 비회원으로 시작

### UX 메모

- 첫 분석 전에는 회원가입 장벽을 낮추는 것이 좋다

## 4-3. 온보딩

### 목적

서비스 사용 원리와 주의사항 설명

### 핵심 요소

- SignalMate가 해주는 것
- 해주지 않는 것
- 채팅 데이터 처리 안내
- 분석 결과는 참고용이라는 디스클레이머

### 주요 액션

- 계속하기

### UX 메모

- 2~3장 이내로 짧아야 한다

## 4-4. 채팅 입력 화면

### 목적

사용자가 분석할 대화를 쉽게 넣도록 한다

### 핵심 요소

- 텍스트 입력 영역
- 붙여넣기 가이드
- 샘플 형식 안내
- 글자 수 카운트
- 민감정보 주의 문구

### 주요 액션

- 채팅 붙여넣기
- 샘플 보기
- 다음으로 이동

### UX 메모

- 카카오톡, 문자, 앱 채팅을 모두 수용할 수 있어야 한다
- 입력 형식이 조금 달라도 너무 쉽게 실패하면 안 된다

## 4-5. 상황 선택 화면

### 목적

분석 해석에 필요한 최소 맥락 입력

### 핵심 요소

- 관계 단계 선택
- 만남 채널 선택
- 현재 고민 선택
- 내 목표 선택

### 주요 액션

- 상황 선택 완료
- 분석 시작

### UX 메모

- 선택지는 많지 않아야 한다
- 사용자가 오래 고민하게 만들면 안 된다

## 4-6. 분석 로딩 화면

### 목적

기다리는 시간을 불안하지 않게 만든다

### 핵심 요소

- 분석 진행 상태 문구
- 어떤 항목을 보고 있는지 간단한 설명
- 저장 여부 안내

### 주요 액션

- 취소
- 분석 계속 대기

### UX 메모

- 단순 스피너보다 `답장 패턴 분석 중`, `약속 신호 정리 중` 같은 문구가 낫다

## 4-7. 분석 결과 요약 화면

### 목적

사용자가 한 번에 전체 분위기를 파악하도록 한다

### 핵심 요소

- 결과 헤드라인
- 긍정 / 애매 / 주의 신호 카운트
- 한 줄 해석
- 지금 추천 액션
- 다음 메시지 보기 버튼
- 근거 상세 보기 버튼

### 주요 액션

- 다음 메시지 추천 보기
- 근거 상세 보기
- 저장하기
- 공유하기

### UX 메모

- 이 화면이 제품의 핵심이다
- 숫자보다 해석 문장이 더 중요하다

## 4-8. 근거 상세 화면

### 목적

사용자가 결과를 납득하게 만든다

### 핵심 요소

- 신호별 설명 카드
- 어떤 패턴을 보고 그렇게 해석했는지 설명
- 확신도 레벨
- 주의할 점

### 주요 액션

- 카드 펼쳐보기
- 결과 저장

### UX 메모

- 모호한 문장보다 구체적인 패턴 언급이 중요하다

## 4-9. 다음 메시지 추천 화면

### 목적

분석을 행동으로 연결한다

### 핵심 요소

- 추천 메시지 3개
- 메시지 톤 가이드
- 피해야 할 표현
- 상황별 추천 이유

### 주요 액션

- 메시지 복사
- 다른 톤으로 다시 추천받기
- 저장하기

### UX 메모

- `예쁘게 쓴 문장`보다 `실제로 보낼 법한 문장`이어야 한다

## 4-10. 저장 분석 목록 화면

### 목적

재방문과 반복 사용을 만든다

### 핵심 요소

- 분석 리스트
- 관계 단계 태그
- 최근 분석 일시
- 요약 결과

### 주요 액션

- 이전 분석 열기
- 재분석하기
- 삭제하기

### UX 메모

- 타임라인 관점이 중요하다

## 4-11. 결제 / 업그레이드 화면

### 목적

무료 사용자에서 유료 사용자로 전환

### 핵심 요소

- 무료와 유료 차이
- 단건 결제 설명
- 구독 혜택 설명
- 개인정보 및 저장 정책

### 주요 액션

- 단건 결제
- 구독 시작

### UX 메모

- `심화 분석`, `메시지 추천`, `저장 기능`을 결제 이유로 묶는 것이 좋다

## 4-12. 설정 / 개인정보 화면

### 목적

신뢰 확보와 저장 정책 제어

### 핵심 요소

- 계정 정보
- 저장 분석 관리
- 데이터 삭제
- 비저장 모드 설명
- 약관 및 개인정보 처리방침 링크

### 주요 액션

- 계정 정보 수정
- 데이터 삭제
- 로그아웃

## 5. MVP 우선순위

### 꼭 필요한 화면

- 랜딩페이지
- 채팅 입력 화면
- 상황 선택 화면
- 분석 로딩 화면
- 분석 결과 요약 화면
- 다음 메시지 추천 화면
- 결제 화면

### 있으면 좋은 화면

- 저장 분석 목록 화면
- 설정 화면
- 근거 상세 화면 분리

MVP에서는 결과 화면 안에 근거 상세를 접어서 넣으면 별도 화면 없이도 시작할 수 있다.

## 6. 핵심 UX 원칙

- 사용자는 빠르게 결과를 보고 싶다
- 결과는 단정이 아니라 해석이어야 한다
- 설명 가능성이 신뢰의 핵심이다
- 결제 전에도 제품 가치를 충분히 체감하게 해야 한다
- 저장과 개인정보 선택권을 명확히 줘야 한다

## 7. 한 줄 결론

SignalMate의 화면 구조는 `입력은 가볍게`, `결과는 설득력 있게`, `다음 행동은 바로 가능하게` 설계하는 것이 핵심이다.
