# SignalMate MVP 기획안

작성일: 2026-03-27

## 1. 제품 한 줄 정의

`SignalMate`는 소개팅, 썸, 초반 연애 단계에서 오가는 채팅 메시지를 바탕으로 관계 신호를 해석하고, 다음 대화를 더 잘 이어갈 수 있게 도와주는 서비스다.

핵심은 `상대의 마음을 단정하는 예언`이 아니라 `채팅 속 관심 신호와 애매한 신호를 구조화해서 보여주는 것`이다.

## 2. 문제 정의

소개팅이나 데이팅 앱을 통해 만난 상대는 공통 지인도 없고, 실제 성향을 알기도 어렵다. 특히 다음과 같은 불확실성이 크다.

- 답장은 오는데 성의가 있는지 모르겠다
- 예의상 대화하는 건지, 실제로 관심이 있는지 헷갈린다
- 한두 번 만난 뒤 대화를 이어가야 할지 판단이 어렵다
- 지금 어떤 메시지를 보내야 관계를 망치지 않을지 모르겠다

사용자는 단순한 `호감도 점수`보다, 지금 대화에서 어떤 신호가 보이는지와 내가 다음에 어떻게 행동하면 좋을지를 원한다.

## 3. 타깃 사용자

### 1차 타깃

- 20대 후반~30대 후반
- 소개팅, 지인 소개, 결혼정보회사, 데이팅 앱 이용 경험이 있는 사람
- 관계 초반의 애매함에서 스트레스를 많이 느끼는 사람

### 2차 타깃

- 연애 경험이 적어 메시지 해석에 자신이 없는 사람
- 소개팅 후속 대화에서 자주 실패한다고 느끼는 사람
- 친구에게 매번 채팅 캡처를 보내며 상담받는 사람

## 4. 고객이 해결하려는 일(Job To Be Done)

- 이 대화가 긍정적인 흐름인지 빠르게 파악하고 싶다
- 상대가 예의상 답하는지, 어느 정도 관심이 있는지 알고 싶다
- 내가 다음에 어떤 톤과 내용으로 답하면 좋을지 추천받고 싶다
- 혼자 감정적으로 해석하지 않고, 근거를 보고 판단하고 싶다

## 5. MVP 가치 제안

MVP는 아래 3가지만 명확히 제공하면 된다.

- `관계 신호 요약`: 긍정, 애매, 주의 신호를 구분해서 보여준다
- `근거 제시`: 어떤 메시지 패턴 때문에 그렇게 판단했는지 설명한다
- `다음 액션 제안`: 다음 메시지 예시와 대화 전략을 준다

## 6. 핵심 사용자 흐름

1. 사용자가 채팅 내용을 붙여넣는다
2. 현재 상황을 짧게 선택한다
3. 서비스가 관계 신호를 분석한다
4. 결과 화면에서 신호 요약, 근거, 위험 포인트를 보여준다
5. 사용자는 다음 메시지 추천을 받는다
6. 필요하면 결과를 저장하거나 며칠 뒤 다시 재분석한다

## 7. MVP 입력 정보

### 필수 입력

- 채팅 메시지 텍스트

### 선택 입력

- 관계 단계: 소개 전 / 첫 만남 전 / 첫 만남 후 / 2~3회 만남 후
- 내 목표: 친해지기 / 약속 잡기 / 애프터 이어가기 / 관계 정리 판단
- 상대와의 만남 채널: 소개팅 / 데이팅 앱 / 결혼정보회사 / 지인 소개

선택 입력은 분석 정확도를 높이기보다 `해석 맥락`을 보정하는 용도로 쓴다.

## 8. MVP 분석 항목

MVP에서는 사용자가 납득할 수 있는 신호 위주로 간다.

- 답장 간격과 변동성
- 상대가 먼저 화제를 확장하는 비율
- 질문을 되묻는 빈도
- 약속이나 일정에 대해 구체적으로 반응하는지
- 대화가 끊겼을 때 다시 이어오려는 시도가 있는지
- 메시지 길이와 톤이 점점 짧아지는지
- 만남 후 후속 메시지의 적극성
- 거절/회피 표현이 반복되는지

## 9. MVP 결과 화면 구성

### A. 신호 요약

- 긍정 신호 3~5개
- 애매 신호 1~3개
- 주의 신호 0~2개

### B. 근거 카드

각 신호에 대해 아래 형식으로 설명한다.

- 어떤 패턴을 봤는지
- 왜 그런 신호로 해석했는지
- 이 해석의 확신도가 높은지 낮은지

### C. 다음 행동 제안

- 지금은 밀어도 되는지, 속도를 늦춰야 하는지
- 다음에 보내면 좋은 메시지 예시 3개
- 피해야 할 메시지 톤 2~3개

## 10. MVP에서 제외할 것

초기 버전에서는 아래 항목을 과감히 제외하는 편이 낫다.

- `상대가 나를 몇 퍼센트 좋아하는지` 같은 단정형 점수
- 음성 톤 분석, 사진 분석, 통화 분석
- 상대 성격 유형 진단
- 혈액형, MBTI, 사주, 별자리를 핵심 판단 근거로 쓰는 기능
- 실시간 메신저 연동

MBTI, 사주, 별자리 등은 추후 `재미 요소`나 `부가 콘텐츠`로는 붙일 수 있지만, MVP의 중심이 되면 신뢰도가 떨어진다.

## 11. 신뢰 설계 원칙

이 서비스는 결과보다 `설명 가능성`이 중요하다.

- 단정 대신 신호 해석으로 표현한다
- 심리 진단처럼 보이는 표현은 피한다
- 분석 근거를 가능한 한 구체적으로 보여준다
- 사용자에게 최종 판단은 본인이 하도록 안내한다

추천 카피 예시:

- `상대의 진짜 마음을 알려드립니다` X
- `채팅 속 관계 신호를 정리해드립니다` O

## 12. 개인정보 및 안전

채팅 데이터는 매우 민감하므로 MVP 단계부터 아래 원칙이 필요하다.

- 업로드 즉시 익명화 안내
- 저장 여부를 사용자가 선택
- 저장하지 않는 일회성 분석 모드 제공
- 민감정보 포함 가능성에 대한 사전 고지
- 결과 화면에 `분석은 참고용`이라는 명확한 디스클레이머 표기

## 13. 성공 지표

MVP에서 볼 핵심 지표는 아래 정도면 충분하다.

- 첫 분석 완료율
- 결과 화면 체류 시간
- 다음 메시지 추천 복사율
- 7일 내 재방문율
- 유료 전환율
- 분석 결과에 대한 주관 만족도

## 14. 초기 수익화 가설

MVP 단계에서는 아래 두 가지를 검증하면 된다.

- 단건 분석 과금이 먹히는가
- 반복 분석을 위해 구독까지 가는 사용자가 있는가

## 15. 4주 MVP 개발 우선순위

### 1주차

- 서비스 콘셉트와 결과 화면 포맷 고정
- 입력 폼, 분석 결과 카드 와이어프레임 제작

### 2주차

- 채팅 붙여넣기 파서
- 관계 신호 분석 로직 1차 구현
- 결과 문구 템플릿 설계

### 3주차

- 다음 메시지 추천 기능
- 저장/재분석 흐름
- 디스클레이머와 개인정보 UX 반영

### 4주차

- 베타 사용자 테스트
- 결과 문구와 가격 실험
- 유료 전환 흐름 점검

## 16. 한 줄 결론

MVP는 `호감 판별기`가 아니라 `채팅 기반 관계 신호 분석 + 다음 액션 코칭 도구`로 시작하는 것이 맞다.
