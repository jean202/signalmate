# SignalMate

작성일: 2026-03-27

## 프로젝트 개요

`SignalMate`는 소개팅, 썸, 연애 초기 단계의 채팅 메시지를 바탕으로 관계 신호를 해석하고, 다음 대화를 어떻게 이어가면 좋을지 제안하는 서비스다.

핵심은 `상대의 마음을 맞히는 것`이 아니라 `채팅 속에서 보이는 관심 신호와 애매한 신호를 구조화해서 보여주는 것`이다.

## 문제 정의

사람들은 관계 초반에 아래 문제를 반복해서 겪는다.

- 답장은 오는데 관심이 있는지 모르겠다
- 예의상 대화하는 건지 실제로 호감이 있는지 헷갈린다
- 첫 만남 후 애프터 가능성을 판단하기 어렵다
- 다음 메시지를 어떻게 보내야 할지 자신이 없다

이미 많은 사용자가 친구에게 채팅 캡처를 보내며 조언을 구하고 있다. SignalMate는 이 행동을 제품으로 정리하는 프로젝트다.

## 제품 포지셔닝

### 하지 않을 것

- 호감도 예언 서비스
- 운세형 연애 판정 앱
- 사주, 혈액형, 별자리 중심 서비스

### 할 것

- 채팅 기반 관계 신호 분석
- 근거 중심 설명
- 다음 메시지 코칭

## MVP 범위 요약

MVP에서는 아래 기능만 명확히 잘하는 것이 중요하다.

- 채팅 텍스트 붙여넣기
- 현재 관계 단계 선택
- 긍정 / 애매 / 주의 신호 분석
- 분석 근거 제시
- 다음 메시지 추천

## 제품 원칙

- 점수보다 근거를 보여준다
- 단정보다 신호 해석을 제공한다
- 연애 예언이 아니라 의사결정 보조 도구로 설계한다
- 개인정보와 채팅 민감도를 최우선으로 본다
- 결과를 행동 제안까지 연결한다

## 추천 MVP 기술 방향

초기 버전은 복잡하게 가지 않는 편이 낫다.

- 프론트엔드: Next.js
- 백엔드: Next.js API Route 또는 Spring Boot
- DB: PostgreSQL
- 캐시/큐: 필요 시 Redis
- 분석 엔진: 규칙 기반 + LLM 보조 설명 생성
- 인증: 이메일 또는 소셜 로그인
- 결제: 국내 결제 모듈 또는 Stripe 대체 검토

초기에는 `정교한 머신러닝 모델`보다 `설명 가능한 규칙 + LLM 요약`이 더 현실적이다.

## 랜딩 프로토타입 메모

`landing-page-nextjs` 폴더에는 아래 항목이 포함되어 있다.

- 랜딩페이지 UI 초안
- `POST /api/v1/waitlist` 라우트
- 로컬 개발용 대기자 등록 저장 로직

현재 대기자 등록은 개발 편의를 위해 로컬 JSON 파일에 저장하는 방식으로 연결되어 있다. 실제 운영으로 갈 때는 PostgreSQL과 Prisma 또는 백엔드 API로 교체하는 것이 맞다.

## 우선 실행 순서

1. 랜딩페이지로 문제 공감과 전환 메시지를 검증한다.
2. 붙여넣기 기반 분석 흐름을 MVP로 구현한다.
3. 결과 화면의 설명력과 메시지 추천 품질을 다듬는다.
4. 단건 결제와 구독 전환을 실험한다.

## 주의할 리스크

- 정확도를 과장하면 신뢰를 잃는다
- 채팅 데이터 저장 방식이 불안하면 이탈한다
- 재미 요소가 메인 가치로 보이면 결제 지속성이 떨어진다

## 한 줄 요약

SignalMate는 `채팅 기반 관계 신호 분석 + 다음 액션 코칭`을 핵심으로 하는 연애 의사결정 도구다.
